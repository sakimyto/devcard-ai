データ導出の実現可能性

GitHub APIから取得できるデータで、4つの情報をどう推定するか

1. 各ツールを何に使っているか

方法: AIコミットのメッセージを分類
feat: → コード生成
fix: → バグ修正
test: → テスト作成
refactor: → リファクタリング
docs: → ドキュメント
精度: Conventional Commits前提。メッセージにprefix無い場合は「その他」に分類

2. ツールごとの使用頻度・比重

方法: Co-Authored-Byのメールで判別
@anthropic.com → Claude
copilot → GitHub Copilot
cursor → Cursor
[bot] → Devin, Sweep等
精度: 高い。Co-Authored-Byが無いAIコミットは「不明」扱い

3. どの言語・領域でAIを使うか

方法: リポジトリの主要言語を取得
GitHub APIの primaryLanguage フィールド
AI設定ファイルがあるリポジトリの言語を集計
精度: リポジトリ単位。ファイル単位の言語判定は不可（GraphQL制約）

4. AIとの協働パターン

方法: コミットパターンから推定
● AI率 70%+ & 連続AI → AI主導
● AI率 30-70% & 交互 → ペアプロ
● AI率 30%未満 → 人間主導
精度: 粗い推定。実際のワークフローは複雑だが、傾向は掴める

問い: この導出方法で気になる点はありますか？
例: 「コミットメッセージがConventional Commitsじゃないケースが多いので、別の判定方法がいい」など
